hexo部署到GitHub

Posted on 2019-07-12

1、注册GitHub，创建免费仓库

bonjour-mark.github.io

SSH授权

ssh-keygen -t rsa

默认三次回车，会在 /root/.ssh/ 目录下生成id_rsa(秘钥)id_rsa.pub(公钥)，将id_rsa.pub公钥的内容复制到GitHub个人设置的SSH keys里

联通测试

ssh -T git@github.com

默认yes，最后会看到成功的提示(开头提示的网络不通请忽略)

创建博客文件

cd /opt/blog/source/_posts
hexo new 'hexo轻博客在RHEL6.5下的安装部署说明'

目录下生成 hexo轻博客在RHEL6.5下的安装部署说明.md 的文件

vi hexo轻博客在RHEL6.5下的安装部署说明.md

将已经写好的markdown格式内容复制到该文件里保存并退出

修改配置文件

cd /opt/blog/
vi _config.yml

将下面的内容复制到该文件最后的deploy：节点保存并退出(注意各分支type、repository、branch冒号后和内容自建要有空格，切记，否则会部署失败)

type: git
repository: git@github.com:bonjour-mark/bonjour-mark.github.io
branch:    master

上传项目

npm install hexo-deployer-git --save
hexo clean
hexo g
hexo d

编译升级gcc-4.8.1.md

Posted on 2019-07-12

升级gcc-4.8.1

下载

cd /opt/docker/
wget http://ftp.tsukuba.wide.ad.jp/software/gcc/releases/gcc-4.8.1/gcc-4.8.1.tar.bz2

解压

tar -jxvf gcc-4.8.1.tar.bz2

下载依赖组件gmp-4.3.2.tar.bz2、mpfr-2.4.2.tar.bz2、mpc-0.8.1.tar.gz

cd gcc-4.8.1
./contrib/download_prerequisites

解压三个包并移到gcc-4.8.1目录

tar -xjf gmp-4.3.2.tar.bz2
tar -xjf mpfr-2.4.2.tar.bz2
tar -xzf mpc-0.8.1.tar.gz
mv gmp-4.3.2 gcc-4.8.1/gmp
mv mpfr-2.4.2 gcc-4.8.1/mpfr
mv mpc-0.8.1 gcc-4.8.1/mpc

创建编译的目录并编译

mkdir gcc-build-4.8.1
cd     gcc-build-4.8.1
../gcc-4.8.1/configure --enable-checking=release --enable-languages=c,c++ --disable-multilib
make -j4
make install

将最新动态库libstdc++.so.6.0.18复制到/usr/lib64目录下

cp /opt/docker/build_gcc_4.8.1/x86_64-unknown-linux-gnu/libstdc++-v3/src/.libs/libstdc++.so.6.0.18 /usr/lib64

进入目录

cd /usr/lib64

删除旧链接

rm -rf libstdc++.so.6

建立新链接

ln -s libstdc++.so.6.0.18 libstdc++.so.6

查询GLIBC的版本

strings /usr/lib64/libstdc++.so.6 | grep GLIBC

升级glibc-2.17

下载并解压

cd /opt/docker/
wget http://ftp.gnu.org/gnu/glibc/glibc-2.17.tar.gz
tar -zxf glibc-2.17.tar.gz

创建编译的目录并编译

cd glibc-2.17
mkdir build
cd build
../configure --prefix=/usr
make -j4
make install

确认glibc版本升级成功

strings /lib64/libc.so.6 |grep GLIBC

hexo在RHEL6.5上的部署说明

Posted on 2019-07-12

1、安装Git

安装Git的依赖包，此步会把低版本的Git安装上，不急，继续往下看

# yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker

获取当前Git版本，通过yum安装的版本太低，自己下载个新一点的版本

# git --version

卸载安装的低版本Git

# yum remove git

进入目录

# cd /opt/docker

下载一个新版本Git

# wget https://github.com/git/git/archive/v2.17.0.tar.gz

解压

# tar -zxvf git-2.17.0.tar.gz

进入目录

# cd git-2.17.0

执行编译

# make prefix=/usr/local/git all

安装Git至/usr/local/git目录

# make prefix=/usr/local/git install

编辑环境变量配置，添加PATH=$PATH:/usr/local/git/bin export PATH 保存

# vim /etc/profile

生效环境变量

# source /etc/profile

获取当前Git版本git version 2.17.0

# git --version

2、安装Nodejs

官网下载对应版本的nodejs文件 https://nodejs.org/en/download/ 将下载的文件上传到RHEL服务器的指定目录，本例中为/opt/docker

进入目录

# cd /opt/docker

解压

# tar -xvf node-v12.6.0-linux-x64.tar.xz

改目录名便于查找

# mv node-v6.10.0-linux-x64 nodejs

建立软连接，变为全局

# ln -s /opt/docker/nodejs/bin/npm /usr/local/bin/
# ln -s /opt/docker/nodejs/bin/node /usr/local/bin/

校验nodejs是否已变为全局

# node -v
# npm -v

安装Hexo

npm install hexo-cli -g

把hexo命令添加到全局

ln -s /opt/docker/nodejs/bin/hexo /usr/local/bin/hexo

查看hexo命令是否正常

cd /
hexo

部署hexo环境

cd /opt/docker/
mkdir blog
cd blog

初始化hexo

hexo init
npm install

启动hexo服务

hexo s

访问hexo

192.168.17.130:4000
